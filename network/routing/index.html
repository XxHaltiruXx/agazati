Routing (Útválasztás)

Mi az a routing?

A routing az a folyamat, amellyel a hálózati eszközök (routerek) meghatározzák a legjobb útvonalat az adatcsomagok számára a forrás és a cél között. A router döntéseit a routing táblázat alapján hozza meg.

Routing alapfogalmak:

  • Router: Hálózati réteg (Layer 3) eszköz, IP címek alapján továbbít
  • Routing Table: Az elérhető hálózatok és útvonalak adatbázisa
  • Next Hop: A következő router IP címe az útvonalon
  • Metric: Az útvonal "költsége" (hop count, bandwidth, delay, stb.)
  • Administrative Distance (AD): Útvonal forrás megbízhatósága (0-255)
Routing táblázat megtekintése:
R1# show ip route

Kódok:
C - Connected (közvetlenül csatlakozó)
S - Static (statikus útvonal)
R - RIP
O - OSPF
D - EIGRP
* - Default route (alapértelmezett útvonal)

Administrative Distance (AD)

Az Administrative Distance meghatározza, hogy a router melyik routing forrásnak hisz jobban, ha ugyanahhoz a célhálózathoz több útvonal is létezik különböző forrásokból.

Routing forrás AD érték Megbízhatóság
Connected interface 0 Legmegbízhatóbb
Static route 1 Nagyon megbízható
EIGRP summary route 5 -
eBGP 20 -
EIGRP (internal) 90 Jó
OSPF 110 Jó
RIP 120 Kevésbé megbízható
EIGRP (external) 170 -
iBGP 200 -
Unknown/Unreachable 255 Nem használt 
Példa:
Ha OSPF (AD=110) és RIP (AD=120) is hirdet egy hálózatot,
a router az OSPF útvonalat választja (alacsonyabb AD = jobb).

Floating static route (backup):
R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.3 150
(AD 150 > OSPF 110, csak akkor aktiválódik, ha OSPF meghal)

Statikus routing

A statikus routing esetén a rendszergazda manuálisan konfigurálja az útvonalakat. Egyszerű, de nem skálázható nagy hálózatokban.

Statikus routing előnyei:

  • Nincs routing protokoll overhead (CPU, RAM, bandwidth)
  • Biztonságosabb (nincs routing hirdetés)
  • Előre látható útvonalak

Statikus routing hátrányai:

  • Nem automatikus, manuális konfiguráció szükséges
  • Nem alkalmazkodik hálózati változásokhoz
  • Nem skálázható nagy hálózatokban
! Statikus útvonal szintaxis
R1(config)# ip route [cél_hálózat] [maszk] [next-hop_IP vagy exit_interface]

! Példák
R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2
R1(config)# ip route 172.16.0.0 255.255.0.0 serial0/0/0

! Default route (alapértelmezett útvonal)
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1

! Floating static route (backup, magasabb AD)
R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.3 150

! Statikus útvonal törlése
R1(config)# no ip route 192.168.2.0 255.255.255.0 10.0.0.2

! Ellenőrzés
R1# show ip route static
R1# show ip route 192.168.2.0

Kétirányú statikus routing példa:

Topológia:
[R1] 192.168.1.0/24 --- 10.0.0.0/30 --- [R2] 192.168.2.0/24

R1 konfiguráció:
R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2

R2 konfiguráció:
R2(config)# ip route 192.168.1.0 255.255.255.0 10.0.0.1

Így mindkét oldal eléri a másik hálózatát.

Dinamikus routing protokollok

A dinamikus routing protokollok automatikusan felépítik és karbantartják a routing táblázatot. A routerek információt cserélnek egymással a hálózati topológiáról.

Routing protokollok osztályozása:

Típus Protokollok Működés
Interior Gateway Protocol (IGP) RIP, OSPF, EIGRP Autonomous System-en belül
Exterior Gateway Protocol (EGP) BGP Autonomous System-ek között

Algoritmus típusok:

Algoritmus Protokollok Működés
Distance Vector RIP, EIGRP (hybrid) Távolság + irány alapján
Link-State OSPF, IS-IS Teljes topológia térkép alapján
Path Vector BGP AS útvonal alapján

RIP (Routing Information Protocol)

A RIP az egyik legrégebbi distance-vector routing protokoll. Egyszerű, de limitált.

RIP jellemzők:

  • Metric: Hop count (maximum 15, 16 = unreachable)
  • Frissítés: 30 másodpercenként broadcast/multicast
  • AD: 120
  • Classful: RIPv1, Classless: RIPv2
  • Multicast cím: 224.0.0.9 (RIPv2)
Jellemző RIPv1 RIPv2
Classful/Classless Classful Classless (CIDR támogatás)
Küldés módja Broadcast (255.255.255.255) Multicast (224.0.0.9)
Subnet maszk Nem küldi Küldi
Hitelesítés Nincs Van (MD5, plaintext)
VLSM támogatás Nem Igen 
! RIPv2 konfiguráció
R1(config)# router rip
R1(config-router)# version 2
R1(config-router)# network 192.168.1.0
R1(config-router)# network 10.0.0.0
R1(config-router)# no auto-summary
R1(config-router)# exit

! Passzív interfész (nem küld RIP frissítéseket)
R1(config-router)# passive-interface gigabitEthernet 0/0

! Default route hirdetése
R1(config-router)# default-information originate

! Ellenőrzés
R1# show ip protocols
R1# show ip rip database
R1# show ip route rip
R1# debug ip rip

RIP problémák és megoldások:

  • Routing loops: Split horizon, route poisoning
  • Count to infinity: Maximum 15 hop limit
  • Lassú konvergencia: Holddown timer (180s)

OSPF (Open Shortest Path First)

Az OSPF egy link-state routing protokoll, amely gyors konvergenciát és skálázhatóságot nyújt. Az egyik legnépszerűbb IGP protokoll.

OSPF jellemzők:

  • Metric: Cost (bandwidth alapján: 10⁸ / bandwidth)
  • Algoritmus: Dijkstra SPF (Shortest Path First)
  • AD: 110
  • Protokoll: IP protokoll 89
  • Multicast címek: 224.0.0.5 (All OSPF routers), 224.0.0.6 (DR/BDR)
  • Hierarchikus: Area rendszer (Area 0 = backbone)

OSPF packet típusok:

Típus Név Funkció
1 Hello Szomszédok felfedezése, keep-alive
2 DBD (Database Description) Link-state adatbázis összefoglalója
3 LSR (Link-State Request) Hiányzó LSA-k kérése
4 LSU (Link-State Update) LSA-k küldése
5 LSAck LSU nyugtázása 
! OSPF alapkonfiguráció
R1(config)# router ospf 1
R1(config-router)# router-id 1.1.1.1
R1(config-router)# network 192.168.1.0 0.0.0.255 area 0
R1(config-router)# network 10.0.0.0 0.0.0.3 area 0
R1(config-router)# exit

! Passzív interfész
R1(config-router)# passive-interface gigabitEthernet 0/0

! Default route hirdetése
R1(config-router)# default-information originate

! Cost módosítása
R1(config)# interface serial 0/0/0
R1(config-if)# ip ospf cost 64
R1(config-if)# exit

! Bandwidth módosítása (hatással van a cost-ra)
R1(config-if)# bandwidth 1544

! Ellenőrzés
R1# show ip ospf
R1# show ip ospf neighbor
R1# show ip ospf interface
R1# show ip ospf database
R1# show ip route ospf

OSPF szomszéd állapotok:

1. Down       - Nincs Hello csomag
2. Init       - Hello csomag érkezett
3. 2-Way      - Kétirányú kommunikáció, DR/BDR választás
4. ExStart    - Master/Slave meghatározás
5. Exchange   - DBD csomagok cseréje
6. Loading    - LSA-k lekérése
7. Full       - Teljes szomszédság (routing információ csere)

DR és BDR (Designated Router, Backup DR):

  • Broadcast és non-broadcast multi-access hálózatokon használt
  • Csökkenti az LSA árasztást
  • Választás prioritás (0-255) és router ID alapján
  • DR/BDR nem preemptív (újraválasztás csak ha valamelyik leáll)
! DR/BDR prioritás beállítása
R1(config)# interface gigabitEthernet 0/0
R1(config-if)# ip ospf priority 100
! (Alapértelmezett: 1, 0 = soha nem lesz DR/BDR)

EIGRP (Enhanced Interior Gateway Routing Protocol)

Az EIGRP egy Cisco saját protokoll (később nyílt szabvánnyá vált). Advanced distance-vector vagy "hybrid" protokoll.

EIGRP jellemzők:

  • Metric: Composite (bandwidth, delay, reliability, load, MTU)
  • Algoritmus: DUAL (Diffusing Update Algorithm)
  • AD: 90 (internal), 170 (external)
  • Protokoll: IP protokoll 88
  • Multicast cím: 224.0.0.10
  • Classless: VLSM támogatás
  • Load balancing: Unequal cost load balancing

EIGRP metric számítás:

Metric = 256 × [(K1 × Bandwidth + (K2 × Bandwidth) / (256 - Load) + K3 × Delay) × K5 / (Reliability + K4)]

Alapértelmezett (K1=1, K2=0, K3=1, K4=0, K5=0):
Metric = 256 × (Bandwidth + Delay)

Bandwidth = 10^7 / bandwidth_legkisebb (Kbps)
Delay = összeg (delay in tens of microseconds)
! EIGRP konfiguráció
R1(config)# router eigrp 100
R1(config-router)# eigrp router-id 1.1.1.1
R1(config-router)# network 192.168.1.0 0.0.0.255
R1(config-router)# network 10.0.0.0 0.0.0.3
R1(config-router)# no auto-summary
R1(config-router)# exit

! Passzív interfész
R1(config-router)# passive-interface gigabitEthernet 0/0

! Hello és Hold timer módosítása
R1(config)# interface serial 0/0/0
R1(config-if)# ip hello-interval eigrp 100 5
R1(config-if)# ip hold-time eigrp 100 15
R1(config-if)# exit

! Unequal cost load balancing
R1(config-router)# variance 2
! (2x-es metrikájú útvonalakat is használ)

! Ellenőrzés
R1# show ip eigrp neighbors
R1# show ip eigrp topology
R1# show ip eigrp interfaces
R1# show ip route eigrp
R1# show ip protocols

EIGRP fogalmak:

  • Successor: Legjobb útvonal a célhoz
  • Feasible Successor (FS): Backup útvonal (loop-free)
  • Feasible Distance (FD): A legjobb útvonal metrikája
  • Reported Distance (RD): Szomszéd által jelentett távolság
  • Feasibility Condition: RD < FD (loop-free garancia)

Routing protokollok összehasonlítása

Jellemző RIP OSPF EIGRP
Típus Distance Vector Link-State Advanced Distance Vector
Metric Hop count Cost (bandwidth) Composite (BW, delay)
AD 120 110 90 (internal)
Max hop/skálázhatóság 15 hop (kicsi) Nincs limit (nagy) 255 hop (nagy)
Konvergencia Lassú Gyors Nagyon gyors
VLSM/CIDR RIPv2: Igen Igen Igen
Load balancing Equal cost Equal cost Equal és unequal cost
CPU/RAM használat Alacsony Közepes-magas Közepes
Szabványosítás Nyílt (RFC 2453) Nyílt (RFC 2328) Cisco (RFC 7868)

Mikor melyiket használjuk:

  • RIP: Kis hálózatok, egyszerű topológiák, tanulási célok
  • OSPF: Nagy vállalati hálózatok, multi-vendor környezet, hierarchikus tervezés
  • EIGRP: Cisco hálózatok, gyors konvergencia szükséges, unequal cost load balancing

Értelmezés:
S* - Statikus default route
   0.0.0.0/0 - Minden cél (default)
   [1/0] - [AD/Metric]
   via 203.0.113.1 - Next hop IP

O  - OSPF tanult
   172.16.0.0/16 - Cél hálózat
   [110/65] - AD 110, Cost 65
   via 10.0.0.2 - Next hop
   00:05:23 - 5 perc 23 másodperce a táblában
   Serial0/0/0 - Kilépő interfész

Útvonal kiválasztási sorrend:

  1. Longest prefix match - Leghosszabb prefix illeszkedés (pl. /30 jobb mint /24)
  2. Administrative Distance - Alacsonyabb AD győz
  3. Metric - Alacsonyabb metric győz (azonos AD esetén)

Route summarization (Route aggregation)

A route summarization több kisebb hálózat összevonása egy nagyobb hálózattá, csökkentve a routing táblázatok méretét és javítva a hatékonyságot. 

Példa: Összevonás
Eredeti hálózatok:
192.168.0.0/24
192.168.1.0/24
192.168.2.0/24
192.168.3.0/24

Bináris elemzés:
192.168.00000000.0
192.168.00000001.0
192.168.00000010.0
192.168.00000011.0

Első 22 bit közös → Összevont: 192.168.0.0/22

! OSPF summarization
R1(config)# router ospf 1
R1(config-router)# area 1 range 192.168.0.0 255.255.252.0

! EIGRP summarization (interfészen)
R1(config)# interface serial 0/0/0
R1(config-if)# ip summary-address eigrp 100 192.168.0.0 255.255.252.0

Summarization előnyei:

  • Kisebb routing táblázatok
  • Gyorsabb SPF számítás (OSPF)
  • Kevesebb frissítés
  • Lokalizált hibák (egy hálózat leállása nem érinti a többi hirdetését)

Routing best practices

  • Használj dinamikus routing protokollt redundáns hálózatokban
  • Válassz egy routing protokollt és maradj annál (ne keverj többet szükségtelenül)
  • OSPF area 0 (backbone) kötelező hierarchikus tervezésnél
  • Passzív interfészek LAN interfészeken (nincs szükség routing hirdetésre)
  • Route summarization nagy hálózatokban
  • Default route használata peremroutereken (internet felé)
  • Floating static routes backup útvonalaknak
  • Router ID manuális beállítása (ne hagyd automatikusra)
  • Hitelesítés routing protokollokban (MD5)
  • Dokumentálás - hálózati diagram, IP címzési terv
! OSPF hitelesítés
R1(config)# interface serial 0/0/0
R1(config-if)# ip ospf message-digest-key 1 md5 SecurePassword
R1(config-if)# ip ospf authentication message-digest
R1(config-if)# exit

! EIGRP hitelesítés
R1(config)# key chain EIGRP_KEY
R1(config-keychain)# key 1
R1(config-keychain-key)# key-string SecurePassword
R1(config-keychain-key)# exit
R1(config-keychain)# exit

R1(config)# interface serial 0/0/0
R1(config-if)# ip authentication mode eigrp 100 md5
R1(config-if)# ip authentication key-chain eigrp 100 EIGRP_KEY
R1(config-if)# exit
